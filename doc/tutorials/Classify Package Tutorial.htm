Classify Package Tutorial

 

The classification package is for using minorthird without text.  This can be useful when you want to classify groups that has a class and a list of features. 

 

You can run three (3) types of experiments in the classify package:

1)      Train –       $ java –Xmx500M edu.cmu.minorthird.classify.Train

a.       Creates a classifier from a trained on a dataset using a learner

2)      Test – j       $ java –Xmx500M edu.cmu.minorthird.classify.Test

a.       Tests the performance of a classifier on a test dataset

3)      TrainTest - $ java –Xmx500M edu.cmu.minorthird.classify.TrainTest

a.       Trains a classifier using a training set and tests the performance on a test dataset, which may be specified or created by a splitter

 

There are three types of experiments that can be run in the classify package: Simple, Seq, and Multi.

1)  Simple - The example below is a sample of simple (non-sequential data).  The b indicates that the class is binary (i.e. it can only be POS or NEG.)  For multi-class data, you will want a k before each line.

 

 

2)  Seq  -  This type of experiment runs on sequential data and uses a sequential learner.  To make the dataset above sequential, you may want to consider whether or not you played tennis the day before.  In this case, you can list the dataset in sequences with a * between each sequence.  For example:

b week1 NEG sunny humid temp=85

b week1 POS sunny humid temp=90

*

b week2 POS sunny dry temp=76

b week2 NEG sunny humid temp = 80

…

            This enables you to consider previous data in each classification.

3)      Multi – A multi experiment is used for examples with multiple labels per example.  For Example: suppose you want to predict whether or not you will play tennis and whether or not you will go swimming on the same day.  Here is a sample dataset for that example, where the first class is for whether you play tennis and the second is for whether you go swimming.

b week1 NEG POS sunny humid temp=85

b week2 POS NEG sunny dry temp=65

            NOTE:  The number of classes per example must be consistent.  Also, you must know the number of classes per example to load the dataset correctly for running an experiment.

 

Choosing the Type of Experiment you will Run:

On the Command Line:

Specify –type [simple,seq,multi]

NOTE:  -type OPTION MUST be specified as the first option on the command line!

 

In the GUI:

Press the “Edit” button next to DataClassificationTask

Select the type of experiment you would like to run from the pull down menu in the Property Editor

 

Experiment Examples:

Each Example goes through what the experiement does, how to run it from the command line and then from the GUI.  Each example is a trainTest experiment.  If you would like to know what parameters you can specify on the command line for any experiment type: $ java –Xmx500M edu.cmu.minorthird.classify.[Train,Test,TrainTest] –help.  For help on any of the parameters definitions, open the GUI and click on the “?” button next to the field.

 

Simple TrainTest

 

Using the command line:

To learn what options are available from the command line type:

% java –Xmx500M edu.cmu.minorthird.classify.TrainTest -help

 

To run an experiment, type this command:

% java –Xmx500M edu.cmu.minorthird.classify.TrainTest –type simple –data simpleData.dat

The output of the program should look like this:

 

Using the gui:

To run an experiment in the gui, first type the command:

% java –Xmx500M edu.cmu.minorthird.classify.TrainTest –gui

 

When the window appears, click the Edit button under Parameter Modification.  This will make the Property Editor window appear.  Select SimpleTrainTestParams from the pull down menu, then press the “Edit” button.

 

 

Next specify the datasetFilename by clicking the Browse button next to the text field.  Find the directory where you saved the file, select the file, and click the Open button. 

 

These next fields have defaults, but can be changed:

  • Learner:  You can specify the learner for this experiment by clicking on the pull down menu next to learnerInSequentialMode NOT next to learner.  Since this is a sequential experiment, only the learner from the learnerInSequentialMode menu will be used.
  • Splitter:  Specify the splitter by choosing one from the pull down menu next to splitter.  You can change any of the splitter options by clicking the Edit button to the right of the pull down menu.

 

Feel free to specify any other parameters.  To view what any of the options do click on the “?” button next to the field.

 

Click the OK button to close the Property Editor and press the “Start Task” button to begin the experiment.

 

When the experiment has finished running, click on the View Results button to see the details of your experiment.

Multi TrainTest

 

Using the command line:

To learn what options are available from the command line type:

% java –Xmx500M edu.cmu.minorthird.classify.TrainTest -help

 

IMPORTANT:  To run a multi experiment you must know the number of classes per example.  Also, the number of classes per example must be consistent.  The number of classes is specified on the command line using the –multi option and in the GUI in the multi field.  Multi MUST be specified before the dataset for the data to load properly!
